### 3. ตัวอย่าง Code สำคัญที่ต้องมี (เพื่อให้ Copilot เขียนตามแนวทางนี้)

#### **A. ส่วนของ Backend (server.js - หัวใจของ G-S-Work)**

ไฟล์นี้จะทำหน้าที่เป็น Agentic Loop จริงๆ ไม่ใช่แค่แชทบอทธรรมดา:

```javascript
// backend/src/server.js
const { GoogleGenerativeAI } = require("@google/generative-ai");
const express = require("express");
const app = express();

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);
const model = genAI.getGenerativeModel({ 
    model: "gemini-2.0-flash", // ใช้รุ่นใหม่ล่าสุด
    tools: [
        { functionDeclarations: [ /* นิยาม toolSearch, toolWriteFile ตรงนี้ */ ] }
    ]
});

app.post("/chat", async (req, res) => {
    const { message, history } = req.body;
    let chat = model.startChat({ history });
    
    // Agentic Loop
    let result = await chat.sendMessage(message);
    let call = result.response.functionCalls();
    
    while(call) {
        // 1. ตรวจสอบว่า AI อยากใช้ Tool อะไร
        // 2. รัน Logic ของ Tool นั้นๆ ใน Backend
        // 3. ส่งผลลัพธ์กลับไปให้ Gemini คิดต่อ
        // 4. วนลูปจนกว่า Gemini จะตอบเป็นข้อความสรุป
    }
    res.json(finalResponse);
});

```

#### **B. ส่วนของ Docker (เพื่อให้ใช้ได้ทั้ง Windows/Mac)**

สร้างไฟล์ `docker-compose.yml` ที่ root folder:

```yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "3000:3000"
    env_file: .env
    volumes:
      - ./workspace:/app/workspace  # พื้นที่ทำงานของ AI Agent

  frontend:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./frontend:/usr/share/nginx/html

```

### 📝 Prompt สำหรับให้ Copilot สร้าง "File Management Tool"

คุณสามารถก๊อปปี้ส่วนนี้ไปสั่ง Copilot เพื่อให้ได้โค้ดที่รองรับ Use Case จัดหมวดหมู่ไฟล์ตามโปรเจกต์ครับ:

> **"ช่วยเขียน Node.js Tool สำหรับ G-A-COWORK เพื่อจัดการไฟล์ในระบบ Docker โดยใช้ความสามารถของ Gemini 2.0 Flash ดังนี้:
> 1. ฟังก์ชัน `list_files(dir)`: ใช้ `fs.readdirSync` เพื่อดูไฟล์ใน `/app/workspace/inbound`
> 2. ฟังก์ชัน `read_document(path)`: ใช้ `pdf-parse` สำหรับ PDF และ `mammoth` สำหรับ Docx เพื่อดึงข้อความออกมาให้ AI วิเคราะห์
> 3. ฟังก์ชัน `organize_project_file(oldPath, projectName, docType, newFileName)`:
> * สร้างโครงสร้างโฟลเดอร์อัตโนมัติ: `/app/workspace/projects/{projectName}/{docType}/`
> * เปลี่ยนชื่อไฟล์ใหม่โดยใส่ Prefix: `[ {projectName} ] {newFileName}`
> * ย้ายไฟล์จากจุดเดิมไปที่โฟลเดอร์ใหม่
> ทั้งหมดนี้ต้องเขียนในรูปแบบ Function Declaration ที่ Gemini เข้าใจได้ (JSON Schema)"**

### 🚀 Master Prompt สำหรับ Copilot (Copy ส่วนนี้ได้เลย)

**Role:** You are an expert Full-stack Developer and AI Engineer.
**Project:** Build an Agentic AI Workspace named **"G-A-COWORK"**.

**Tech Stack:**

* **Backend:** Node.js (Express), Google Generative AI SDK (Gemini 2.0 Flash).
* **Frontend:** Single Page HTML (use the design from `agentic-ai-workspace.html`), Tailwind CSS, Lucide Icons.
* **Environment:** Docker & Docker Compose.
* **Tools:** `pdf-parse` (for PDF), `mammoth` (for Docx), `tavily-client` (for Web Search).

**Requirements:**

1. **Agentic Loop:** Implement a robust `while` loop in the backend that handles Gemini's `functionCalls`. The agent must be able to call multiple tools in sequence until the task is finished.
2. **File Management System (Crucial):**
* Create a `/workspace` directory mapped via Docker volume.
* Tool `list_files`: List all files in `/workspace/inbound`.
* Tool `read_document`: Extract text from PDF, Docx, or TXT.
* Tool `organize_file`:
* Input: `oldPath`, `projectName`, `docType`, `newFileName`.
* Logic: Create folder `/workspace/projects/{projectName}/{docType}/` if it doesn't exist.
* Action: Rename file to `[{projectName}] {newFileName}` and move it to the new folder.




3. **Trace Visibility:** The frontend must display every step of the "Thinking Process" (e.g., "AI is reading scan_01.pdf...", "AI identified this as Project Alpha Invoice").
4. **Dockerization:** - Provide a `Dockerfile` for the backend.
* Provide a `docker-compose.yml` that sets up the Backend, a simple Nginx Frontend, and the `/workspace` volume.



**Files to Generate:**

1. `backend/package.json` (with dependencies: `@google/generative-ai`, `express`, `pdf-parse`, `mammoth`, `dotenv`, `cors`).
2. `backend/server.js` (The heart of G-A-COWORK with the Agentic Loop).
3. `backend/tools.js` (Definition and logic for all tools).
4. `docker-compose.yml`.
5. `.env.example` (Fields: `GEMINI_API_KEY`, `TAVILY_API_KEY`).
